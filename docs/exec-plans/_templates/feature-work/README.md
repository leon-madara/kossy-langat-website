# Feature Work Template

Copy this folder to:

`docs/exec-plans/active/<feature-slug>/`

Use it for any new feature, animation system, multi-session build, or cross-page UI change that needs continuity between sessions or between agents.

## Required Files

- `README.md` - feature overview, scope, links, and current status
- `plan.md` - execution phases and sequence of work
- `todo.md` - next tasks, backlog, blockers, and active checklist
- `done.md` - completed work log
- `decisions.md` - accepted, rejected, and deferred decisions
- `handoff.md` - end-of-session status for the next agent
- `verification.md` - checks run, results, and outstanding verification
- `assets-prompts.md` - image, asset, and prompt direction when applicable

## Suggested Setup Steps

1. Copy the folder to `docs/exec-plans/active/<feature-slug>/`.
2. Replace all placeholder values.
3. Fill in `README.md` and `plan.md` before implementation starts.
4. Keep `todo.md`, `done.md`, `decisions.md`, and `handoff.md` current during the feature.
5. Record checks in `verification.md`.
6. Add prompt and asset notes in `assets-prompts.md` if image generation is part of the feature.
