# Decisions - Home Construction Philosophy Sequence

## Accepted

- 2026-03-09 - Decision: add the requested extra dwell time as a dedicated final hold segment instead of stretching the whole pinned narrative. Why: the user wants more time to read the last line and study the last frame, not slower pacing across every beat. Impact: the main frame scrub still uses the original `250vh` pacing, and only the release into Gap is delayed by an extra `10vh`.
- 2026-03-09 - Decision: Philosophy pin ownership must not wait on frame readiness. Why: when the pin registered only after the first frame became renderable, the Hero -> Philosophy boundary could fall through normal document flow for a frame on slower loads. Impact: the section now pins immediately, while the loader and first-frame readiness gate only rendering/text sync.
- 2026-03-09 - Decision: fix the Hero -> Philosophy -> Gap flicker by stabilizing Philosophy startup instead of adding more Gap motion. Why: Gap-local GSAP reduced symptoms but left the wrong owner in the handoff and made the boundary harder to reason about. Impact: Gap is static again and the boundary now has one scroll-motion owner.
- 2026-03-08 - Decision: keep Hero on the original fixed-image handoff pattern while removing only the image lift and darkening pass. Why: the requested behavior is the earlier content fade plus blur-to-clear reveal, not a fully static copy layer. Impact: Hero now keeps the image fixed, fades the copy, clears the blurred overlay, and still remains self-contained and independent from downstream sections.
- 2026-03-08 - Decision: initialize Philosophy on the first successfully loaded frame and keep the section live while the rest of the sequence settles. Why: waiting for all 192 frames delayed pin registration and made slow-load behavior structurally unstable. Impact: the pin now registers early, the loader becomes informational instead of blocking, and the canvas can catch up as more frames arrive.
- 2026-03-08 - Decision: use overlay-specific foreground tokens for mobile Philosophy copy instead of inheriting the section's light-theme foreground directly. Why: the mobile presentation still uses a dark readability scrim even in light theme, so the normal light-theme foreground turned the copy too dark. Impact: mobile and auto/light-theme modes now keep readable light copy over the lower-third scrim without changing the desktop surface palette.
- 2026-03-08 - Decision: degrade Philosophy to a static narrative when reduced motion is requested or no sequence frames load. Why: the section still needs to communicate the narrative when motion or imagery is unavailable. Impact: users keep a readable non-pinned experience instead of a blank or invisible sequence state.
- 2026-03-08 - Decision: do not use document-level native scroll snap for Gap. Why: there is no local scroll container in the `Hero -> Philosophy -> Gap` region, so native snap would have to attach to the page scroller and could interfere with pinned sections and downstream micro-pins. Impact: Gap stays static and any remaining settle behavior will be owned by the Philosophy exit instead.
- 2026-03-08 - Decision: opt `PhilosophySequence` out of `ScrollMicroPin` locally via `data-micro-pin="off"` instead of adding a route-specific exception in `ScrollMicroPin.tsx`. Why: this keeps pin ownership declared at the section boundary and avoids home-page branching inside the global pin system. Impact: Philosophy owns its own pin while the rest of the site keeps the shared micro-pin behavior.
- 2026-03-08 - Decision: treat the current code changes as a Phase 1 checkpoint, not the full Hero or Philosophy rebuild. Why: boundary cleanup needed to land and verify before pacing and art-direction tuning. Impact: the next work stays focused on the Gap settle choice, then the planned Hero and Philosophy rebuild phases.
- 2026-03-08 - Decision: rewrite the `Hero -> Philosophy -> Gap` flow around independent animation ownership. Why: the old hero-to-next-section coupling no longer fits the page order. Impact: Hero stops animating the next section, Philosophy owns the only pin in this area, and Gap becomes a static reading section.
- 2026-03-08 - Decision: Gap should not receive its own GSAP animation. Why: the desired behavior is a readable section that passes naturally, not another motion stage. Impact: any settle behavior must be owned by the Philosophy exit, not by Gap-local GSAP.
- 2026-03-08 - Decision: preserve `FeaturedProjects` behavior during this rewrite unless integration breaks. Why: this refactor is about fixing the upstream section architecture, not redesigning downstream motion. Impact: verification still includes the transition into `FeaturedProjects`, but implementation scope remains focused.
- 2026-03-08 - Decision: move the feature back out of signoff mode and treat it as active debug work. Why: runtime checks found structural issues that are more serious than minor art-direction tweaks. Impact: continuity docs now prioritize fixing pin ownership and mobile readability before any closeout.
- 2026-03-08 - Decision: treat the global `ScrollMicroPin` interaction as the first debugging target. Why: the philosophy section is already using its own long-form `pin: true` ScrollTrigger, and GSAP warns that nested pinning is not supported. Impact: micro-pin must be removed or explicitly disabled for this section before tuning pacing.
- 2026-03-08 - Decision: treat mobile light-theme readability as a blocking bug, not a polish item. Why: the design note requires a stronger scrim for readability on mobile, but the current light-theme token swap makes the text too dark against that scrim. Impact: theme-specific copy contrast must be corrected before user review resumes.
- 2026-03-07 - Decision: place the sequence between Hero and GapProblem. Why: it keeps the narrative arc identity -> philosophy -> problem. Impact: home route order and section numbering changed.
- 2026-03-07 - Decision: use canvas sequence rendering for 192 frames. Why: lower DOM overhead and tighter scroll-frame coupling than swapping many image elements. Impact: dedicated preload/draw logic is required.
- 2026-03-07 - Decision: keep GSAP ScrollTrigger pin/scrub as the core interaction model. Why: project standards require GSAP for timeline/scroll interactions. Impact: all sequence motion tuning stays in GSAP timelines.
- 2026-03-07 - Decision: move this feature into stabilization mode. Why: baseline implementation exists but timeline and layout issues still affect quality. Impact: plan phases now prioritize correctness over net-new scope.
- 2026-03-07 - Decision: normalize sequence timing with explicit frame tween duration and normalized text times. Why: ensures frame progression and text choreography span the full pinned timeline consistently. Impact: pinned scroll now has deterministic timing behavior.
- 2026-03-07 - Decision: reset canvas transform before applying DPR scale on resize and redraw current frame. Why: prevents cumulative scale artifacts and keeps frame rendering stable after viewport changes. Impact: resize/orientation transitions remain visually correct.
- 2026-03-07 - Decision: implement true desktop side-rail markup (text rail sibling to canvas stage) with mobile scrim overlay. Why: aligns code with intended layout and improves readability on image-heavy frames. Impact: desktop composition now reads as side-rail instead of accidental overlay.
- 2026-03-07 - Decision: express the pin range as an explicit `250vh` viewport-based distance. Why: the prior percentage-based end value was both longer than requested and less predictable across viewport sizes. Impact: scroll pacing now matches the requested duration more directly.
- 2026-03-07 - Decision: let GSAP own philosophy-line transforms and opacity states entirely. Why: CSS base transforms plus GSAP `y` tweens were producing jitter, double-rendering, and reverse-scroll jumps. Impact: desktop rail motion is now stable and deterministic.
- 2026-03-07 - Decision: use soft crossfades on desktop but discrete line swaps on mobile. Why: mobile line lengths and font scale make any overlap unreadable, while desktop benefits from a softer handoff. Impact: one line remains readable at all times on mobile without sacrificing desktop polish.
- 2026-03-07 - Decision: render mobile frames in cover mode and anchor copy in the lower third. Why: contain-mode created large letterboxing gaps and detached the text from the imagery. Impact: mobile now presents as a full-bleed narrative stage instead of a centered image with floating copy.
